Repositories

A scoville-repo has two tasks: providing modules and providing templates. The scoville-cores talk to the repository to obtain their resources. In your scoville infrastructure you will need to register at least one repository. You have two possibilities to do that. Either register an existing repository or create your own.

Register an existing repository

This is a little unconvenient until now, because this feature has not been needed during development (will be fixed). First obtain the IP of the server the scoville-repo runs on. Then add this IP as a server. Right mouseclick on the newly entered server. Click Properties.... Click on Add in the Instances-Frame. Select the InstanceType ScovilleRepository. Enter the URL of the repiository into URL you can also enter IPs here like http://10.10.10.10. If you are an administrator of this scoville-repo you may enter your password into the field Password. Now you have a repository.

Set up an own repository

Setting up an own is a bit tricky until now. First, generate a debian-package of scoville-repo from the sources by executing the following commands:
user@scoville-admin:~$ cd /path/to/scoville/repo/install
user@scpville-admin:~$ ./generate_packages.sh
Now scp the generated .deb-file to the machine you want to create your scoville-repo on.
user@scoville-admin:~$ scp scoville-repo.deb user@machine:/tmp/
After that SSH into the machine and install the package
user@scoville-admin:~$ ssh user@machine
user@scoville-repo:~$ cd /tmp
user@scoville-repo:~$ sudo su
root@scoville-repo:~# dpkg -i scoville-repo.deb
root@scoville-repo:~# apt-get -f -y install
Follow instructions on screen. You can now call the URL that points to your scoville-repo-Machine and should be able to see the machines public key. Cońtinue with the section above to register your new repository with scoville-admin.
